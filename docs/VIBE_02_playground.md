# VibeCoder Playground — 재미 요소 기획

> **핵심 문제**: 바이브코더는 막히는 순간 포기한다.
> 터미널 에러, 낯선 명령어, 혼자라는 느낌. 이 세 가지가 포기의 원인이다.
> 이 기획은 그 순간마다 개입해서 "여기 오면 해결된다"는 경험을 만드는 것이다.

- **타겟**: 코알못 바이브코더 (코딩 몰라도 AI로 앱 만들고 싶은 사람)
- **컨셉**: 놀이터 — 실패해도 괜찮고, 작은 성공도 자랑할 수 있는 곳
- **방향**: 막히면 해결해주고, 성공하면 같이 기뻐해주는 사이트

---

## 기능 개요 및 우선순위

| 순위 | 기능명 | 한 줄 설명 | 임팩트 | 난이도 |
|------|--------|-----------|--------|--------|
| 🥇 1 | **에러 응급실** | 에러 붙여넣기 → 한국어 해결책 | ★★★★★ | ⭐⭐⭐ |
| 🥈 2 | **명령어 레시피북** | 상황별 명령어 복사 모음 | ★★★★ | ⭐ |
| 🥉 3 | **자랑 게시판** | 만든 것 공유, 칭찬만 있는 공간 | ★★★★ | ⭐⭐ |
| 4 | **오늘의 챌린지** | 매일 30분 미션, 재방문 유도 | ★★★ | ⭐⭐⭐ |
| 5 | **바이브 레벨** | 활동 기반 레벨/뱃지 시스템 | ★★★ | ⭐⭐⭐⭐ |

## P0 보안/안정성 게이트 (오픈 전 필수)

아래 항목을 만족하기 전까지 `에러 응급실`과 관련 API를 외부 오픈하지 않는다.

- [ ] 입력 안전
  - 사용자 입력 최대 2,000자 제한
  - 시스템 프롬프트와 사용자 입력을 분리해 전달
  - 프롬프트 인젝션 패턴("위 지시 무시")을 명령 생성 근거로 사용하지 않음
- [ ] 위험 명령어 통제
  - `rm -rf`, `sudo`, `curl|bash`, `chmod 777` 등은 기본 해결 단계에서 노출 금지
  - 위험 명령어는 `plan_b`에서만 노출 + 기본 숨김 + 2단계 확인
  - 가능한 경우 비파괴 대체 명령 우선 제시
- [ ] 장애 fallback
  - Gemini 실패/타임아웃 시 에러 타입별 정적 템플릿 응답 반환
  - 빈 응답/빈 화면 금지 (항상 다음 행동 안내 포함)
- [ ] 남용 방지
  - `/api/error-translate`, 박수 API에 IP 기반 rate limit 적용
  - 임계치 초과 시 captcha 또는 동등 검증 단계 적용

## Feature 1. 에러 응급실 🚑

### 왜 이게 가장 중요한가

```
바이브코더의 포기 패턴:
아이디어 → AI로 코딩 → 실행 → 빨간 에러
   ↓
에러 복사 → 구글링 → 영어 답변 → 이해 못 함
   ↓
포기 → 바이브코딩 그만둠
```

에러 응급실은 이 포기 직전 순간에 개입한다.
Gemini API가 이미 연동되어 있으니 추가 비용도 없다.

### 사용자 플로우

```
1. 터미널 에러 메시지 복사
2. 에러 응급실 입력창에 붙여넣기
3. [번역하기] 클릭
4. 결과:
   ├── 📋 무슨 일이냐면 (한국어 설명)
   ├── 🔧 이렇게 해보세요 (단계별 해결책, 복사 버튼 포함)
   └── 😱 그래도 안 되면 (플랜 B)
5. "해결됐어요!" 버튼 → 해결 사례 DB에 축적
```

### UI 설계

```
┌──────────────────────────────────────────────────┐
│  🚑 에러 응급실                                   │
│  터미널에서 빨간 글씨가 떴나요? 여기 붙여넣으세요. │
├──────────────────────────────────────────────────┤
│                                                  │
│  [에러 메시지를 여기에 붙여넣으세요...]            │
│                                                  │
│                            [번역하기 →]           │
├──────────────────────────────────────────────────┤
│  📋 무슨 일이냐면:                                │
│  파일을 찾을 수 없어요. node_modules 폴더가       │
│  없거나 깨진 상태예요.                            │
│                                                  │
│  🔧 이렇게 해보세요:                              │
│  ① 아래 명령어를 터미널에 복사해서 실행하세요      │
│  ┌────────────────────┐                          │
│  │ pnpm install       │  [복사]                  │
│  └────────────────────┘                          │
│                                                  │
│  😱 그래도 안 되면:                               │
│  ┌────────────────────────────────────┐          │
│  │ rm -rf node_modules && pnpm install │  [복사] │
│  └────────────────────────────────────┘          │
│                                                  │
│  [✅ 해결됐어요!]   [❌ 아직도 안 돼요]           │
└──────────────────────────────────────────────────┘
```

### Gemini 프롬프트 설계

```python
ERROR_PROMPT = """
다음은 바이브코딩 중 발생한 터미널 에러야.
코딩을 전혀 모르는 사람도 이해할 수 있게 설명해줘.

에러:
{error_message}

다음 형식으로 JSON만 응답해:
{
  "what_happened": "왜 이런 에러가 났는지 2줄 이내, 기술 용어 사용 금지",
  "fix_steps": [
    {"description": "무엇을 하는 명령어인지", "command": "실행할 명령어"}
  ],
  "plan_b": {
    "description": "위 방법이 안 될 때",
    "command": "대안 명령어"
  },
  "error_type": "pnpm|python|git|vite|general"
}

원칙:
- 기술 용어 쓰지 말 것 (예: 'dependency' → '필요한 파일들')
- 위험 명령어는 기본 단계에 노출하지 말 것 (`plan_b` 기본 숨김)
- 설명은 초등학생도 이해할 수 있게
"""
```

### 해결 사례 DB 활용

```sql
CREATE TABLE error_solutions (
  id            SERIAL PRIMARY KEY,
  error_hash    TEXT,              -- 에러 유형 해시 (유사 에러 그룹화)
  error_type    TEXT,              -- pnpm | python | git | vite | general
  error_excerpt TEXT,              -- 에러 앞 200자
  solution      JSONB,             -- Gemini 생성 해결책
  solved_count  INTEGER DEFAULT 0, -- "해결됐어요" 클릭 수
  failed_count  INTEGER DEFAULT 0, -- "안 돼요" 클릭 수
  created_at    TIMESTAMP DEFAULT NOW()
);
```

- 같은 에러 재입력 시 → DB 캐시 활용 (Gemini 호출 절약)
- 해결률 높은 에러+해결책 → 레시피북에 자동 추가

## Feature 2. 명령어 레시피북 📖

### 컨셉

요리 레시피처럼 "이럴 때 이거 쓰면 돼"를 모아놓은 정적 페이지.
복잡한 문서 읽을 필요 없이, 상황만 찾으면 명령어가 바로 나온다.

### 카테고리 구성

```
📖 명령어 레시피북

🆕 프로젝트 새로 시작할 때
🔥 서버 켜고 끄기
📦 패키지 설치/삭제
🐛 에러 났을 때 (에러 응급실 연동)
🌿 Git 기본 (저장하고 올리기)
🚀 배포할 때
💀 전부 꼬였을 때 (핵옵션)
```

### 레시피 카드 예시

```
┌─────────────────────────────────────────┐
│ 🔥 서버 켜기                             │
│                                         │
│ ┌─────────────────────┐                 │
│ │  pnpm dev           │  [복사]         │
│ └─────────────────────┘                 │
│                                         │
│ [이게 뭐야?]                            │
│ ↓ 클릭하면 펼쳐짐                       │
│ "로컬 컴퓨터에서 내 사이트를 미리 보는   │
│  명령어예요. 켜면 브라우저에서 확인 가능" │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ 💀 전부 꼬였을 때 (핵옵션)              │
│                                         │
│ ┌────────────────────────────────────┐  │
│ │ rm -rf node_modules && pnpm install │ [복사] │
│ └────────────────────────────────────┘  │
│                                         │
│ ⚠️ 이건 node_modules 폴더를 통째로      │
│ 지우고 다시 설치해요. 시간이 좀 걸려요.  │
└─────────────────────────────────────────┘
```

## Feature 3. 자랑 게시판 🏆

### 컨셉

"나도 만들었어요!" — 판단 없이 칭찬만 있는 공간.
완성된 앱이 아니어도 된다. 버튼 하나 바꾼 것도 자랑할 수 있다.

바이브코딩의 핵심 동기부여는 **"나도 할 수 있다"**는 확신이다.
다른 사람의 작은 성공이 그 확신을 만든다.

### 게시물 구조

```
┌──────────────────────────────────────────┐
│ 🚀 오늘 배포 성공했어요!                 │
│ by 김바이브 · 3시간 전                   │
│                                          │
│ [스크린샷 또는 링크 미리보기]            │
│                                          │
│ 🛠️ 사용 도구: Cursor + Claude            │
│ ⏱️ 걸린 시간: 이틀                       │
│ 😅 제일 어려웠던 것: 로그인 만들기       │
│ 💡 이 툴 최고: Cursor 자동완성           │
│                                          │
│ 🏷️ #첫배포  #React  #초보자             │
│                                          │
│ 👏 243   💬 12   🔖 저도 만들어볼게요   │
└──────────────────────────────────────────┘
```

### 자동 뱃지 시스템

```
게시물 내용 분석 → 자동 뱃지 부여

🌱 첫 게시물       → "첫 발걸음" 뱃지
🚀 "배포" 포함     → "배포 성공" 뱃지
🐛 "에러" "해결" 포함 → "에러 극복" 뱃지
⏱️ 걸린 시간 < 1시간 → "스피드 바이브" 뱃지
👥 댓글 10개 이상  → "화제의 작품" 뱃지
```

```
│  오늘 187명 도전 중 🔥   완료율 64%     │

└──────────────────────────────────────────┘
```

### 챌린지 DB 구조

```sql
CREATE TABLE challenges (
  id          SERIAL PRIMARY KEY,
  level       TEXT,              -- beginner | mid | expert
  title       TEXT NOT NULL,     -- "버튼 색깔 바꾸기"
  description TEXT NOT NULL,     -- 상세 설명
  hint        TEXT,              -- 힌트
  tools       TEXT[],            -- ['Cursor', 'Claude']
  display_date DATE UNIQUE,      -- 노출 날짜
  created_at  TIMESTAMP DEFAULT NOW()
);

CREATE TABLE challenge_completions (
  id           SERIAL PRIMARY KEY,
  challenge_id INTEGER REFERENCES challenges(id),
  user_id      INTEGER REFERENCES users(id),
  completed_at TIMESTAMP DEFAULT NOW()
);
```

## Feature 5. 바이브 레벨 시스템 🎮

### 컨셉

활동하면 경험치가 쌓이고 레벨이 오른다.
레벨보다 **뱃지 수집**이 핵심 — 각 뱃지가 하나의 성취 스토리.

### 경험치 획득 방법

| 활동 | 경험치 |
|------|--------|
| 에러 응급실 사용 후 "해결됐어요" | +10 XP |
| 자랑 게시판 첫 게시물 | +50 XP |
| 오늘의 챌린지 완료 | +20 XP |
| 챌린지 7일 연속 완료 | +100 XP 보너스 |
| GitHub 카드 북마크 | +5 XP |
| 댓글 달기 | +5 XP |
| 다른 사람 게시물에 👏 | +2 XP |

### 레벨 체계

```
Lv.1   🌱 씨앗              (0 ~ 99 XP)
Lv.2   🌿 새싹              (100 ~ 299 XP)
Lv.3   🌳 나무              (300 ~ 599 XP)
Lv.4   🗺️ 탐험가            (600 ~ 999 XP)
Lv.5   🔨 건축가            (1,000 ~ 1,999 XP)
Lv.6   🚀 런처              (2,000 ~ 3,999 XP)
Lv.7   ⚡ 바이브 마스터      (4,000 XP ~)
```

### 뱃지 목록

```
성취 뱃지:
🌱 첫 발걸음      — 첫 로그인
🚀 첫 배포        — 자랑 게시판에 "배포" 게시물
🐛 에러 극복      — 에러 응급실 5회 해결
📖 레시피 마스터  — 레시피북 10개 즐겨찾기
🔥 불꽃 스트릭    — 챌린지 7일 연속
💪 30일 도전      — 챌린지 30일 완료
👥 팀플레이어     — 댓글 10개 달기
💡 아이디어 뱅크  — 게시물 5개 올리기
⭐ 인기스타       — 게시물 박수 100개
🇰🇷 국산 응원단   — 국내 개발자 레포 10개 북마크
```

## 구현 권장 순서

```
1단계 (빠른 임팩트, 1~2주)
  ├── 명령어 레시피북  ← 가장 쉽고 즉각 유용
  └── 에러 응급실     ← Gemini 연동 이미 있음

2단계 (커뮤니티 형성, 3~4주)
  └── 자랑 게시판     ← 기존 프로젝트 게시판 확장

3단계 (습관 형성, 5~6주)
  └── 오늘의 챌린지   ← 콘텐츠 사전 준비 필요

4단계 (장기 락인, 7~8주)
  └── 바이브 레벨 시스템 ← 1~3단계 완성 후 얹기
```

## 핵심 원칙

1. **막힌 순간에 개입한다** — 포기하기 직전이 골든타임
2. **판단하지 않는다** — 작은 성공도 자랑할 수 있는 공간
3. **기술 용어 없음** — 모든 UI 텍스트는 코알못 기준
4. **즉시 사용 가능** — 로그인 없이 에러 응급실, 레시피북 이용 가능
5. **같이 하는 느낌** — 혼자가 아니라는 것을 계속 보여줌
