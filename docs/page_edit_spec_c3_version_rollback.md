# C-3 버전 비교/롤백 명세

## 목적

버전 이력을 신뢰 가능하게 비교하고, 안전하게 롤백할 수 있는 절차를 정의한다.

## 문서 상태

- 상태: Draft v1
- Sprint: 3
- 선행: A-2, C-2, C-1a
- 후행: E-1

## 버전 비교

- 비교 단위: 블록 추가/삭제/순서 변경/필드 변경
- MVP 비교 방식:
  - 텍스트/핵심 필드 diff 우선
  - 구조적 시각 diff는 후속

## 롤백 절차

1. 대상 version 선택
2. diff 검토
3. reason 입력
4. rollback 실행 (`POST /rollback`)
5. 복원된 draft 검증 후 publish 여부 결정

## 안전장치

- `super_admin`만 rollback 허용
- reason 필수
- rollback 결과는 새 version으로 생성(기존 이력 불변)

## 실패/복구

- 대상 버전 없음: 404
- 권한 없음: 403
- 서버 오류: 500 + 재시도 안내
