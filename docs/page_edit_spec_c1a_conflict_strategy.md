# C-1a 충돌 감지 전략 명세

## 목적

다중 편집 시 충돌을 예측 가능하게 처리한다.

## 문서 상태

- 상태: Draft v1
- Sprint: 2
- 선행: C-1, A-2
- 후행: C-3

## 전략 후보

### 전략 A: 낙관적 동시성 (권장)

- 저장 시 `baseVersion` 비교
- 불일치면 409 반환
- 장점: 구현 단순, 락 운영 부담 낮음
- 단점: 사용자가 충돌 해결 과정을 이해해야 함

### 전략 B: 편집 락

- 페이지별 편집 세션 락 획득
- 만료/강제해제 정책 필요
- 장점: 충돌 빈도 감소
- 단점: 운영 복잡도 및 데드락 관리 부담

## MVP 결정

- 기본: 전략 A 채택
- 전략 B는 후속 Phase 4 검토

## 충돌 UX

- 충돌 발생 시:
  - 내 변경 임시 보관
  - 최신 버전 diff 링크 제공
  - 재적용(merge assist) 옵션은 후속 범위
