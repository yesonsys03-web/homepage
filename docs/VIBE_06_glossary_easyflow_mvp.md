## 2) /glossary 화면 정보 구조 (코알못 우선)

순서 원칙: 용어 설명보다 "지금 어떤 상황인지"를 먼저 보여준다.

1. 상단 히어로: "오늘 막힘 방지 3개"
2. 상황 필터: "지금 내가 뭘 하려는지"
3. 명령어 카드 리스트: 쉬운 설명 + 복사
4. 다음 행동 CTA: 런치패드/레시피 이동

### 2-1. 화면 섹션 맵

```text
/glossary
┌──────────────────────────────────────────────┐
│ Hero: 오늘 막힘 방지 3개                      │
├──────────────────────────────────────────────┤
│ 상황 필터: 처음 시작 / 실행 / 에러 / 배포      │
├──────────────────────────────────────────────┤
│ 추천 명령어 카드 리스트                        │
│ - 한 줄 뜻                                   │
│ - 쉽게 말하면                                │
│ - 이럴 때 써요                               │
│ - 복사 / 관련 용어 / 런치패드 CTA             │
├──────────────────────────────────────────────┤
│ 연결 섹션: 관련 용어 더 보기 / 레시피 링크     │
└──────────────────────────────────────────────┘
```

### 2-2. 컴포넌트 레벨 매핑

- 상단 추천 영역: `TodayGlossaryCards` 확장 또는 전용 상단 컴포넌트
- 상황 필터: `GlossaryScreen` 상단 필터 영역 확장
- 명령어 카드: 기존 glossary 카드의 정보 우선순위 재배치
- 연결 CTA: `런치패드에서 따라하기`, `관련 용어 보기`, `레시피 보기`

## 3) 화면 카피 초안 (바로 사용 가능)

### 3-1. 히어로 영역

- 타이틀: `오늘 막힘 방지 3개`
- 서브카피: `지금 당장 자주 쓰는 명령어만 골랐어요. 외우지 말고, 필요한 순간에 꺼내 써요.`
- 보조 문구: `명령어가 무서우면 정상이에요. 여기서는 한 줄씩 이해하고 복사하면 됩니다.`

버튼:
- 기본: `3개 먼저 보기`
- 보조: `용어 전체 보기`

### 3-2. 상황 필터(탭) 카피

- `처음 시작할 때`
- `실행할 때`
- `에러 났을 때`
- `배포할 때`
- `고급/주의`

필터 안내 문구:
- `상황을 고르면, 지금 필요한 명령어만 보여줘요.`

### 3-3. 명령어 카드 고정 포맷

카드 헤더:
- 상황 배지 예: `처음 시작할 때`
- 위험도 배지: `안전`, `주의`, `위험`

카드 본문 순서:
1. 한 줄 뜻: `이 명령어가 하는 일`
2. 생활 비유: `쉽게 말하면`
3. 언제 쓰는지: `이럴 때 써요`
4. 실행 줄: 코드 블록 + 복사 버튼

하단 액션:
- `복사하기`
- `왜 필요한지 보기`
- `관련 용어 보기`
- `런치패드에서 따라하기`

### 3-4. 위험 명령어 가이드 카피

- 배지 문구:
  - `안전`: `실수해도 복구가 쉬워요`
  - `주의`: `실행 전 폴더/대상을 꼭 확인하세요`
  - `위험`: `파일이 사라질 수 있어요. 백업 후 진행하세요`

- 위험 명령 카드 상단 경고:
  - `이 명령어는 데이터 삭제 가능성이 있어요. 초보자는 가이드 없이 실행하지 마세요.`

### 3-5. 빈 상태/검색 실패 카피

- 빈 상태: `아직 이 상황의 추천 명령어를 준비 중이에요.`
- 검색 실패: `같은 뜻의 쉬운 단어로 다시 검색해보세요. 예: 설치, 실행, 오류`
- 재시도 CTA: `오늘 추천으로 돌아가기`

## 4) 명령어 카드 예시 카피

### 예시 A: `pnpm install`

- 한 줄 뜻: `프로젝트에 필요한 준비물을 한 번에 설치해요.`
- 쉽게 말하면: `요리 시작 전에 장바구니 재료를 한 번에 사오는 단계예요.`
- 이럴 때 써요: `코드를 처음 받았거나 node_modules가 비어 있을 때`
- 버튼: `복사하기`

### 예시 B: `pnpm dev`

- 한 줄 뜻: `내 컴퓨터에서 개발 서버를 켜요.`
- 쉽게 말하면: `가게 셔터를 올리고 손님을 맞을 준비를 하는 거예요.`
- 이럴 때 써요: `화면을 수정하고 바로 결과를 확인하고 싶을 때`

### 예시 C: `uv sync`

- 한 줄 뜻: `파이썬 프로젝트에 필요한 환경을 맞춰요.`
- 쉽게 말하면: `작업실 공구함을 설명서대로 다시 채우는 단계예요.`
- 이럴 때 써요: `의존성 오류가 나거나 처음 환경을 맞출 때`

## 6) LLM 자동 수집 UX 반영 규칙 (MVP)

초기 목표는 자동 "발행"이 아니라 자동 "초안 수집"이다.

- 표시 원칙:
  - `오늘 새로 들어온 용어`는 상단에서 3개만 노출
  - 상태가 `pending`이면 관리자 화면에서만 전체 노출
  - 사용자 화면은 `active`만 노출

- 품질 게이트(최소):
  - `term` 중복 금지(대소문자/공백 정규화)
  - `one_liner` 15자 이상
  - `analogy` 30자 이상
  - 위험 명령어 포함 시 자동 `주의/위험` 라벨

## 7) 상호작용 규칙

### 7-1. 기본 상호작용

- 카드 전체 클릭보다 버튼 역할을 분리한다.
- `복사하기`는 가장 눈에 띄는 1차 CTA로 둔다.
- `왜 필요한지 보기`는 설명 확장 토글로 처리한다.
- `관련 용어 보기`는 같은 화면 내 포커스 이동으로 처리한다.
- MVP에서는 180도 카드 플립을 기본 상호작용으로 넣지 않는다.
- 이유는 명령어 실행과 다음 행동 CTA를 즉시 보이게 유지하는 것이 더 중요하기 때문이다.

### 7-2. 런치패드 연결 규칙

- `런치패드에서 따라하기` 클릭 시 탭 전환 + 관련 블록 자동 포커스
- 같은 명령어가 여러 가이드에 걸치면 가장 초보자 친화적인 가이드를 우선 노출

### 7-3. 모바일 규칙

- 모바일에서는 코드 줄 길이를 줄이고 복사 버튼을 크게 둔다.
- 모바일에서는 `터미널은 PC에서 실행` 안내 배너를 상단에 유지한다.
