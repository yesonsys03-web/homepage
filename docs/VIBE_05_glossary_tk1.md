# 바이브 용어사전: 터미널 스킬 트리 도입 기획안 (VIBE_05_glossary_tk1)

> **핵심 가치**: "터미널은 외우는 게 아니라 흐름으로 익히는 것"
> LLM을 활용해 코알못이 제일 무서워하는 **터미널 명령어** 사전을 자동 확장하고, 서로 연관된 개념을 "스킬 트리" 형태로 묶어주는 스마트 비유 사전 구축.

- **위치**: `/glossary` 화면 내 "터미널", "기본개념" 등 기존 탭 활용
- **목표**: 
  1. 관리자 개입 없이 매일 3개씩 신규 터미널/코딩 용어 자동 갱신
  2. 단순 나열을 벗어나 **상황별 흐름**과 **연관 명령어** 기반의 꼬리물기 UX 제공

---

## 1. 구현 방향 및 아키텍처 (비용 $0)

현재 구축된 백엔드 인프라(FastAPI Background Tasks + Gemini API)를 재활용하여 **자동화 파이프라인**을 구성합니다.

### 1-1. 매일 3건 자동 생성 파이프라인
* **스케줄러**: `server/main.py`에 적용된 `run_periodic_async_loop`를 이용해 매일 특정 시각(예: 자정)에 백그라운드 태스크 실행.
* **AI 생성 (Gemini 1.5 Flash)**: 
  * 현재 DB(`glossary_terms` 테이블)에 등록된 명령어 목록을 프롬프트 컨텍스트로 주입.
  * **프롬프트 내용**: "초보 바이브코더가 알면 좋은 새로운 터미널 명령어/도구 3개를 찾아, 중학생 수준의 비유와 '관련 명령어(related)'를 묶어 JSON 배포해 줘."
* **저장**: 생성된 3건은 DB에 즉시 Insert되거나 임시 상태(pending)로 저장되어 관리자 승인 후 표시.

### 1-2. DB 스키마 검토/확장
기존의 `glossary_terms` 테이블 구조가 대부분 일치하나, 추후 스킬 트리나 흐름을 위해 일부 속성 추가 검토.
* 필드: `id`, `term`, `one_liner`, `analogy`, `when_appears`, `category`, `related` (문자열 배열)
* 자동 생성 여부를 나타내는 플래그 필요: `is_auto_generated` (boolean), `status` (pending | active)

## 2. 사용자 시나리오 (UX)

단어장의 "단어 하나 보기"가 아니라, **문제 해결을 위한 개념 지도 탐험**에 초점을 맞춥니다.

### 2-1. 해시태그 기반 "꼬리에 꼬리를 무는 네비게이션"
- 사용자가 `npm install` 용어를 열어봄.
- **관련 용어 태그 노출**: 카드 하단에 연관된 태그가 표시됨 (#package.json, #npm run dev, #node_modules)
- **자연스러운 전이**: 클릭 시 해당 용어 카드로 부드럽게 필터링/포커스 전환되며, 개념 사이의 연결고리를 체감.

### 2-2. 상황별(Workflow) 그룹핑 모드 추가
알파벳/이름순 정렬 이외에, 프론트엔드 카테고리 필터(`selectedCategory`)에 "작업 흐름" 기반 묶음 제공:
- **[📦 프로젝트 처음 셋업할 때]**: `git clone`, `npm install`, `uv sync`
- **[🚀 실행하고 테스트할 때]**: `npm run dev`, `python main.py`, `pnpm build`
- **[🐛 꼬였을 때 다 밀어버리기]**: `rm -rf node_modules`, `pnpm store prune`

### 2-3. "오늘 새로 배운 스킬" 대시보드
- `/glossary` 화면 상단 (현재 `TodayGlossaryCards` 컴포넌트 위치) 활용.
- 백그라운드 스케줄러가 방금 막 생성해온 **따끈따끈한 AI 자동 생성 용어 3개**를 하이라이트 표시.
- 매일매일 새로운 지식이 추가되는 활력을 줌.

## 4. 재미 요소 (Fun Factors) — 게이미피케이션 도입

지루한 학습이 아니라 "놀이터"처럼 즐길 수 있는 장치들을 마련합니다.

### 4-1. 터미널 스킬 & 뱃지 시스템 (XP 연동)
- **내용**: 특정 카테고리의 용어를 필달(조회/복사)하면 전용 뱃지 부여.
- **예**: `cd`, `ls` 등 기본 명령어 마스터 시 "터미널 뉴비" 뱃지 하사.
- **효과**: `xp_service.py`와 연동하여 사용자의 성장 척도를 시각화.

### 4-2. 최고의 비유 투표 (Analogy World Cup)
- **내용**: AI가 생성한 비유가 마음에 들면 "이게 진짜 찰떡임! 👍" 버튼 클릭.
- **효과**: 사용자 반응이 좋은 비유는 상단에 고정하거나 "명예의 전당" 카드로 별도 노출.

### 4-3. 가상 터미널 시뮬레이터 (VIBE Sandbox 🧪) - **강력 추천**
- **비주얼 인터랙션**: 명령어 입력 시 옆의 가상 파일 탐색기(Folder Tree)가 실시간으로 생성/삭제/이동하는 애니메이션 노출 (예: `mkdir` 시 폴더 생성을 시각화).
- **안전한 실수(Safe Fail)**: 파일 삭제(`rm`) 등 위험한 명령어를 가상 공간에서 마음껏 실행해보고 시각적 파괴 효과와 복구 과정을 통해 구조를 이해 (공포감 → 제어 가능성).
- **퀘스트 모드**: "프로젝트 폴더를 만들고 그 안으로 들어가 보세요!" 같은 가이드 미션을 주고 성공 시 XP 및 전용 뱃지 부여 (Small Win 제공).
- **테마 커스텀**: 사용자의 취향에 맞는 예쁜 터미널 스킨 제공 (애플 Sleek, 파스텔, 매트릭스 등 테마로 애착 형성).

### 4-4. "오늘의 요정" - 페르소나 선택
- **내용**: 지식 설명 톤을 사용자가 직접 선택 (예: 친절한 형, 츤데레 선배, 동화 선생님).
- **효과**: Gemini가 선택된 페르소나에 맞춰 비유와 설명을 다시 써줌으로써 반복적인 탐색의 재미 제공.

### 4-5. 3D 카드 플립 (Magic Card Flip) - **인터랙션 강화**
- **내용**: 용어 카드를 클릭하면 180도 회전(Flip)하며 뒷면의 상세 내용을 노출.
- **구성**:
  - **앞면(Front)**: 용어명, 짧은 요약, 카테고리 태그 (깔끔한 카드 디자인).
  - **뒷면(Back)**: AI가 쓴 비유, 연관 명령어 버튼, **[VIBE Sandbox 🧪]** 진입 버튼.
- **효과**: 실물 단어장을 넘기는 듯한 타격감과 재미를 주며, 정보의 양을 효과적으로 분리하여 인지 부하 감소.

### 4-6. Playground 레시피북 연동 (Recipe Link) - **실전 연결**
- **내용**: 용어사전과 Playground의 '명령어 레시피북'을 양방향으로 연결하여 "이론 → 실습" 흐름 완성.
- **기능**:
  - **사전 → 레시피**: 특정 용어(예: `Git`)의 사전 카드 뒷면에 "관련 레시피 보기" 버튼 추가. 클릭 시 Playground의 해당 카테고리로 이동.
  - **레시피 → 사전**: Playground 레시피 설명 문구에 `GlossaryHighlightedText` 적용. 레시피를 읽다 모르는 단어(예: `의존성`, `빌드`)가 나오면 바로 툴팁으로 확인.
- **효과**: "이 단어가 실제 어떤 명령어에 쓰이는지"를 즉시 확인하게 함으로써 학습의 실용성 극대화.

### 4-7. 이모지 코드 해부학 (Emoji Anatomy) - **직관적 이해**
- **내용**: 복잡한 명령어 예시의 각 부분에 직관적인 이모지를 매핑하여 시각적으로 설명.
- **예시**: `pnpm` 📦(택배), `install` 🚚(배송), `package.json` 📜(주문서)
- **효과**: 텍스트 읽기를 힘들어하는 코알못들이 코드를 "그림"으로 인지하고 빠르게 기능을 이해하게 함.

### 4-8. 바이브 광장 전광판 (VIBE Live Ticker) - **커뮤니티 온기**
- **내용**: 화면 하단에 작고 예쁜 띠 배너를 통해 현재 다른 유저들의 활동을 실시간(혹은 의사 실시간)으로 노출.
- **예시**: "누군가 `Docker`의 찰떡 비유를 보고 감탄했습니다! 💡", "현재 인기 용어: `Cursor`, `pnpm`"
- **효과**: 놀이터에 혼자가 아니라는 느낌을 주며, 다른 이들의 학습 경로를 보고 자연스럽게 호기심 유발.

## 5. 구현 순서 (마일스톤)

### Phase 1: 자동화 기반 마련 (서버 & DB)
1. **DB 업데이트**: `glossary_terms` 테이블에 `is_auto_generated`, `status` 컬럼 추가.
2. **AI 프롬프트 작성**: `server/gemini_curator.py` 혹은 신규 유틸리티에 매일 용어 3건 생성용 프롬프트 로직 작성.
3. **스케줄러 등록**: `server/main.py` 시작 시 실행되는 백그라운드 태스크에 하루 단위 주기 등록.

### Phase 2: 연결형 탐색 UX 및 재미 요소 도입 (프론트)
1. **Magic Card Flip & Navigation**: 카드 회전 인터랙션 구현 및 연관 단어 클릭 시 부드러운 이동/포커스 로직.
2. **Playground & Recipe Integration**: 레시피북과 사전 간의 양방향 링크 및 하이라이트 툴팁 적용.
3. **VIBE Sandbox 🧪**: 가상 터미널 환경 구축 및 시각적 애니메이션 연동.
4. **Emoji Anatomy & Ticker**: 코드 예시 이모지 매핑 시스템 및 하단 라이브 Ticker 구현.

## 4. 확장 아이디어 (미래)
- 유저들이 "이 명령어는 도움이 되었어요!" 투표(Clap) 진행 시, 인기 명령어들은 스킬 트리 내에서 "필수 마스터 노드" 뱃지를 얻음.
- 명령어가 위험한 것(예: 파일 삭제)일 경우 AI가 자동으로 `CAUTION` 등급 발급.
