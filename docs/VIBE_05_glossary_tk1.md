# 바이브 용어사전: 터미널 스킬 트리 도입 기획안 (VIBE_05_glossary_tk1)

> **핵심 가치**: "터미널은 외우는 게 아니라 흐름으로 익히는 것"
> LLM을 활용해 코알못이 제일 무서워하는 **터미널 명령어** 사전을 자동 확장하고, 서로 연관된 개념을 "스킬 트리" 형태로 묶어주는 스마트 비유 사전 구축.

- **위치**: `/glossary` 화면 내 "터미널", "기본개념" 등 기존 탭 활용
- **목표**: 
  1. 관리자 개입 없이 매일 3개씩 신규 터미널/코딩 용어 자동 갱신
  2. 단순 나열을 벗어나 **상황별 흐름**과 **연관 명령어** 기반의 꼬리물기 UX 제공

---

## 1. 구현 방향 및 아키텍처 (비용 $0)

현재 구축된 백엔드 인프라(FastAPI Background Tasks + Gemini API)를 재활용하여 **자동화 파이프라인**을 구성합니다.

### 1-1. 매일 3건 자동 생성 파이프라인
* **스케줄러**: `server/main.py`에 적용된 `run_periodic_async_loop`를 이용해 매일 특정 시각(예: 자정)에 백그라운드 태스크 실행.
* **AI 생성 (Gemini 1.5 Flash)**: 
  * 현재 DB(`glossary_terms` 테이블)에 등록된 명령어 목록을 프롬프트 컨텍스트로 주입.
  * **프롬프트 내용**: "초보 바이브코더가 알면 좋은 새로운 터미널 명령어/도구 3개를 찾아, 중학생 수준의 비유와 '관련 명령어(related)'를 묶어 JSON 배포해 줘."
* **저장**: 생성된 3건은 DB에 즉시 Insert되거나 임시 상태(pending)로 저장되어 관리자 승인 후 표시.

### 1-2. DB 스키마 검토/확장
기존의 `glossary_terms` 테이블 구조가 대부분 일치하나, 추후 스킬 트리나 흐름을 위해 일부 속성 추가 검토.
* 필드: `id`, `term`, `one_liner`, `analogy`, `when_appears`, `category`, `related` (문자열 배열)
* 자동 생성 여부를 나타내는 플래그 필요: `is_auto_generated` (boolean), `status` (pending | active)

## 2. 사용자 시나리오 (UX)

단어장의 "단어 하나 보기"가 아니라, **문제 해결을 위한 개념 지도 탐험**에 초점을 맞춥니다.

### 2-1. 해시태그 기반 "꼬리에 꼬리를 무는 네비게이션"
- 사용자가 `npm install` 용어를 열어봄.
- **관련 용어 태그 노출**: 카드 하단에 연관된 태그가 표시됨 (#package.json, #npm run dev, #node_modules)
- **자연스러운 전이**: 클릭 시 해당 용어 카드로 부드럽게 필터링/포커스 전환되며, 개념 사이의 연결고리를 체감.

### 2-2. 상황별(Workflow) 그룹핑 모드 추가
알파벳/이름순 정렬 이외에, 프론트엔드 카테고리 필터(`selectedCategory`)에 "작업 흐름" 기반 묶음 제공:
- **[📦 프로젝트 처음 셋업할 때]**: `git clone`, `npm install`, `uv sync`
- **[🚀 실행하고 테스트할 때]**: `npm run dev`, `python main.py`, `pnpm build`
- **[🐛 꼬였을 때 다 밀어버리기]**: `rm -rf node_modules`, `pnpm store prune`

### 2-3. "오늘 새로 배운 스킬" 대시보드
- `/glossary` 화면 상단 (현재 `TodayGlossaryCards` 컴포넌트 위치) 활용.
- 백그라운드 스케줄러가 방금 막 생성해온 **따끈따끈한 AI 자동 생성 용어 3개**를 하이라이트 표시.
- 매일매일 새로운 지식이 추가되는 활력을 줌.

## 3. 구현 태스크 쪼개기 (마일스톤)

### Phase 1: 자동화 기반 마련 (서버 & DB)
1. **DB 업데이트**: `glossary_terms` 테이블에 `is_auto_generated`, `status` 컬럼 추가.
2. **AI 프롬프트 작성**: `server/gemini_curator.py` 혹은 신규 유틸리티에 매일 용어 3건 생성용 프롬프트 로직 작성.
3. **스케줄러 등록**: `server/main.py` 시작 시 실행되는 백그라운드 태스크에 하루 단위 주기 등록.

### Phase 2: 연결형 탐색 UX 도입 (프론트)
1. 기존 `GlossaryScreen.tsx`에서 연관 단어(`related`) 클릭 시 부드럽게 스크롤되며 해당 단어를 하이라이트 하거나 검색어를 자동 변경하는 로직 개선.
2. "상황별" 그룹핑 UI 디자인 반영.
3. 자동 생성된 용어가 Admin 없이 바로 표시되게 할지, Admin의 승인을 거치게 할지 (UI: 뱃지 처리 고민 등) 결정 및 백엔드 연동.

## 4. 확장 아이디어 (미래)
- 유저들이 "이 명령어는 도움이 되었어요!" 투표(Clap) 진행 시, 인기 명령어들은 스킬 트리 내에서 "필수 마스터 노드" 뱃지를 얻음.
- 명령어가 위험한 것(예: 파일 삭제)일 경우 AI가 자동으로 `CAUTION` 등급 발급.
