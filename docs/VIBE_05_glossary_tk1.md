# 바이브 용어사전 전략 문서: 터미널 명령어 자동 수집과 연결형 탐색 (VIBE_05_glossary_tk1)

> 핵심 가치: "터미널은 외우는 게 아니라 흐름으로 익히는 것"
> 전략 목표: 코알못이 무서워하는 터미널 명령어를 LLM으로 지속 수집하되, 안전하게 검수하고, 상황 기반 탐색으로 연결한다.

- 위치: `/glossary` 화면 내 기존 용어사전 경험 확장
- 역할: 자동 수집, 데이터 모델, 운영 정책, 단계별 로드맵을 정의하는 상위 전략 문서
- UX 실행 문서: `docs/VIBE_06_glossary_easyflow_mvp.md`
- 실습 연결 문서: `docs/VIBE_04_launchpad.md`

---

## 1. 왜 지금 이걸 하는가

현재 글로서리의 가장 큰 한계는 "정적 데이터"다.
운영자가 직접 채워야 해서 확장 속도가 느리고, 코알못이 실제로 막히는 최신 명령어 흐름을 빠르게 반영하기 어렵다.

이 문서의 목표는 단순히 용어 수를 늘리는 것이 아니다.
핵심은 아래 3가지를 동시에 만드는 것이다.

1. 필요한 명령어가 계속 들어오는 구조
2. 초보자에게 위험하지 않게 보여주는 운영 구조
3. 용어 -> 실행 -> 문제 해결로 이어지는 제품 구조

---

## 2. 제품 원칙

### 2-1. 상황 우선

- 사용자는 명령어 이름보다 "언제 쓰는지"를 먼저 이해해야 한다.
- 따라서 명령어는 항상 상황 맥락과 함께 노출한다.

### 2-2. 자동 생성보다 안전한 활성화

- LLM의 역할은 자동 발행이 아니라 자동 초안 생성이다.
- 사용자에게 노출되는 데이터는 반드시 `active` 상태만 허용한다.

### 2-3. 단어장보다 흐름

- 목표는 용어 정의집이 아니라, 막힐 때 따라갈 수 있는 흐름 지도다.
- related, workflow, launchpad 연결은 모두 이 원칙 아래에서 움직인다.

### 2-4. 위험 명령어 분리

- 삭제/초기화 계열 명령어는 일반 추천 목록에서 전면 노출하지 않는다.
- 별도 `고급/주의` 영역에서 강한 경고와 함께 다룬다.

---

## 3. 이번 전략의 문제 정의

코알못은 아래에서 가장 크게 막힌다.

1. 무슨 명령어를 쳐야 하는지 모른다.
2. 명령어를 봐도 왜 필요한지 모른다.
3. 위험한 명령어와 안전한 명령어를 구분하지 못한다.
4. 용어사전과 실제 실행 공간이 분리되어 있어 학습이 끊긴다.

이 전략은 위 4개를 풀기 위해 설계한다.

---

## 4. 전략 범위

### 4-1. 이번 문서에서 다루는 범위

- LLM 기반 터미널/코딩 용어 자동 초안 수집
- 용어 데이터 모델과 상태 정책
- related 기반 연결 탐색
- workflow 기반 그룹핑 전략
- Launchpad 연동 원칙
- 단계별 구현 우선순위

### 4-2. 이번 문서에서 바로 구현 대상으로 보지 않는 범위

- 3D 카드 플립
- 실시간 전광판
- 가상 터미널 샌드박스
- 페르소나별 설명 재작성
- 대규모 게이미피케이션 확장

이 요소들은 가치가 없어서가 아니라, 핵심 루프가 안정화되기 전에는 운영 복잡도만 키우기 쉽다.

---

## 5. 핵심 전략

### 5-1. 자동 수집 전략

- 매일 3개의 신규 용어 또는 명령어 초안을 생성한다.
- 생성 대상은 초보자 가치가 높은 항목으로 제한한다.
- 출력 형식은 구조화된 JSON으로 고정한다.

생성 기준 예시:
- 바이브코딩 입문자가 실제로 자주 마주치는 명령어
- 설치, 실행, 동기화, 배포, 오류 복구에 필요한 기본 명령어
- 단일 명령어뿐 아니라 꼭 같이 배워야 하는 연관 개념 포함

### 5-2. 검수 전략

- 자동 생성 결과는 기본적으로 `pending` 상태로 저장한다.
- 검수 기준을 통과한 항목만 `active`로 전환한다.
- 운영 초기에 무검수 자동 발행은 하지 않는다.
- 사용자에게 보이는 명령어 카드 내용은 어드민이 쉽게 직접 수정할 수 있어야 한다.

### 5-3. 탐색 전략

- 용어 하나를 읽고 끝나는 구조가 아니라, related 클릭으로 다음 카드로 이동한다.
- 카테고리 정렬 외에 workflow 관점의 묶음을 제공한다.
- 추천 목록은 "상황 해결" 기준으로 보여준다.

### 5-4. 학습 연결 전략

- Glossary는 "왜 이 명령어가 필요한지" 설명한다.
- Launchpad는 "어디서 어떻게 입력하는지"를 안내한다.
- 이 둘은 같은 `/glossary` 경험 안에서 왕복 가능해야 한다.

## 6. 운영 정책

### 6-1. 상태 정책

- `pending`: LLM 초안 생성 완료, 사용자 비노출
- `active`: 검수 완료, 사용자 노출 가능
- `rejected`: 품질/중복/위험도 문제로 폐기
- `archived`: 과거 항목 보관, 기본 목록 비노출

### 6-2. 품질 정책

최소 기준:
- term 중복 금지
- one_liner 최소 길이 확보
- analogy 최소 길이 확보
- category 강제
- related 최대 개수 제한

권장 추가 기준:
- alias 또는 slug 보유
- model/prompt version 기록
- 생성 일시와 검수 일시 기록
- 마지막 수정자와 수정 시각 기록

### 6-3. 위험 명령어 정책

- `rm`, 초기화, 전역 삭제, 권한 변경 계열은 별도 분류한다.
- 일반 추천 탭에는 노출하지 않는다.
- 사용자 노출 시 반드시 `주의` 또는 `위험` 배지를 부착한다.

---

## 7. 데이터 모델 방향

초기 필수 필드:
- `id`
- `term`
- `one_liner`
- `analogy`
- `when_appears`
- `category`
- `related`
- `status`
- `is_auto_generated`

추가 권장 필드:
- `slug`
- `aliases`
- `risk_level`
- `workflow_tags`
- `source_model`
- `prompt_version`
- `created_at`
- `reviewed_at`

중요 판단:
- `related`는 장기적으로 단순 문자열보다 안정 키 기반이 낫다.
- term 문자열만 키로 쓰면 rename, alias, 오탈자 처리에서 깨질 가능성이 높다.

---

## 8. 사용자 경험 전략 요약

상세 화면 카피와 배치는 `docs/VIBE_06_glossary_easyflow_mvp.md`에서 정의한다.
이 문서에서 고정하는 전략 판단은 아래와 같다.

1. 화면 최상단에는 "오늘 막힘 방지 3개"를 둔다.
2. 사용자는 카테고리보다 상황 필터를 먼저 본다.
3. 카드 정보 순서는 "한 줄 뜻 -> 쉬운 비유 -> 언제 쓰는지 -> 명령어"다.
4. 각 카드는 Launchpad 또는 레시피로 이어지는 다음 행동을 가진다.

---

## 9. Launchpad 연결 전략

`docs/VIBE_04_launchpad.md`와의 역할 분담은 아래와 같다.

- Glossary: 개념 이해, 용어 연결, 상황 맥락 설명
- Launchpad: 실제 복사/붙여넣기, 환경 준비, 에러 해결

핵심 동선:
1. 사용자가 `/glossary`에서 명령어 카드를 본다.
2. 카드에서 `런치패드에서 따라하기`를 누른다.
3. 같은 페이지 안의 런치패드 탭 또는 해당 실습 블록으로 이동한다.

## 10. 단계별 로드맵

### Phase 1. 데이터와 운영 기반

목표:
- LLM 자동 초안 수집 구조 확립
- 상태 기반 검수 흐름 확보

포함 항목:
1. glossary_terms 계열 데이터 모델 정리
2. 매일 3건 생성 잡 등록
3. pending 저장 및 기본 검수 기준 정의

### Phase 2. 핵심 탐색 경험

목표:
- 코알못이 무섭지 않게 탐색하고 복사할 수 있는 경험 제공

포함 항목:
1. 상황 기반 필터
2. related 기반 연결 이동
3. 오늘 막힘 방지 3개 노출
4. Launchpad 연결 CTA

### Phase 3. 학습 강화 요소

목표:
- 반복 방문과 학습 전환을 높이는 가벼운 장치 추가

포함 항목:
1. 비유 투표
2. 배지/XP 최소 연동
3. 도움됨 피드백

### Phase 4. 확장 실험

후속 검토 대상:
- Sandbox
- Persona
- Magic Card Flip
- Live Ticker

판단 메모:
- `Magic Card Flip`은 시각적으로 매력적이지만, MVP 단계에서는 `복사하기`와 `런치패드에서 따라하기`를 숨기지 않는 것이 더 중요하다.
- 따라서 플립 인터랙션은 핵심 학습 루프가 안정화된 뒤 붙이는 상호작용 강화 항목으로 본다.

## 11. 성공 판단 기준

### 정량 지표

- 오늘 추천 카드 클릭률
