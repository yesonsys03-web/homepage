# Page Edit UX 개선안 Take 2

## 1) 문서 목적

현재 페이지 편집 메뉴에서 `블록 목록 / 캔버스 미리보기 / 속성 패널` 배치가 비효율적이라, 특히 속성 패널 접근을 위해 스크롤이 필요한 문제를 해소하기 위한 기획안이다.

본 문서는 **구현 지시가 아닌 UX 설계 기준**이며, 목표는 다음 두 가지다.

- 패널 탐색 비용 제거(찾기 위해 스크롤하지 않도록)
- 편집 집중 흐름 강화(한 번에 하나의 작업 영역에 집중)

---

## 2) 현재 문제 요약

현행 편집 탭은 화면 폭 구간에 따라 2열/3열로 동작한다.

- 데스크톱 넓은 구간에서는 3열(블록 목록, 캔버스, 속성 패널)
- 그보다 좁은 구간에서는 실질적으로 2열 기반 레이아웃 + 속성 패널이 아래로 밀리는 상황이 발생
- 결과적으로 사용자는 아래로 스크롤해 속성 패널을 찾아야 하며, 블록 선택 -> 속성 수정 흐름이 끊긴다

사용자 관점의 핵심 불편:

1. 속성 패널이 즉시 보이지 않음
2. 블록 선택 후 편집까지 이동거리가 큼
3. 캔버스 확인과 속성 수정 왕복이 비효율적

---

## 3) 제안 핵심: Side Rail + Single Visible Panel

### 3.1 구조 원칙

편집 화면 **좌측 고정 영역**에 **사이드 레일(Side Rail)** 을 두고, 그 아래에 다음 3개 항목을 배치한다.

- 블록 목록
- 캔버스 미리보기
- 속성 패널

레일에서 항목을 선택하면, 본문 영역에는 **선택한 항목 1개만 표시**한다.

즉, 기존의 "여러 패널 동시 노출 + 스크롤 탐색"에서
"항목 선택 기반 단일 패널 노출"로 전환한다.

### 3.2 접기/펼치기

사이드 레일 상단에 `접기/펼치기` 버튼을 둔다.

- 펼침 상태: 아이콘 + 라벨 노출
- 접힘 상태: 아이콘만 노출(툴팁으로 라벨 제공)

접힘 상태에서도 현재 활성 항목은 시각적으로 명확히 표시한다.

### 3.3 레일 폭 규칙 (고정 수치)

- 펼침(Expanded): 기본 `176px` (최소 `160px`, 최대 `220px`)
- 접힘(Collapsed): `48px` (아이콘 레일)
- 레일 리사이즈는 Desktop/Notebook에서만 허용하고, Mobile에서는 리사이즈를 제공하지 않는다.
- 리사이즈 핸들은 레일의 우측 엣지에 고정한다.
- 리사이즈 중 본문 패널은 실시간 재배치(reflow)한다.
- 리사이즈 폭은 `localStorage`(`page_editor_rail_width`)에 저장하고 재진입 시 복원한다.

## 4) 인터랙션 설계

### 4.1 기본 진입 상태

- 기본 활성 항목: `블록 목록`
  - 이유: 첫 진입 시 "무엇을 편집할지 선택"이 선행되어야 하며, 빈 속성 패널 노출을 피한다.
- 블록이 선택되기 전 속성 패널은 `No selection` 안내 상태를 유지한다.
- 블록이 하나도 없으면 본문에 Setup empty-state를 노출한다.
  - 메시지: "편집할 블록을 먼저 추가하세요"
  - 액션: "블록 추가"

### 4.2 패널 전환 규칙

- 레일 항목 클릭 시 즉시 전환
- 전환 시에도 아래 상태는 유지
  - 선택된 블록
  - 입력 중 값(미저장 상태 포함)
  - 캔버스 스크롤 위치

### 4.3 보조 단축 흐름

- 블록 목록에서 블록 클릭 -> 기본적으로 속성 패널 자동 전환(`ON` 기본값)
- 캔버스에서 블록 클릭 -> 기본적으로 속성 패널 자동 전환(`ON` 기본값)
- 자동 전환 `OFF`는 운영 설정(고급 옵션)에서만 허용한다.

### 4.4 미저장 상태 전환 정책 (Dirty State)

- 같은 편집 세션 내 패널 전환은 입력값을 유지하며, 즉시 폐기하지 않는다.
- 아래 상황에서 Dirty 상태이면 확인 모달을 강제한다.
  - 다른 블록으로 선택 변경
  - 편집 탭 이탈
  - 페이지 새로고침/탭 닫기
- 확인 액션은 `저장 후 이동 / 폐기 후 이동 / 취소` 3가지만 허용한다.
- 시스템은 미저장 값을 조용히 버리지 않는다.

## 5) 반응형 동작 기준

### 5.1 Desktop (>= 1440)

- `Single Visible 모드`를 기본값으로 사용한다.
- Desktop에서도 3패널 병행 패러다임을 유지하지 않는다(설계 단일화).
- 필요 시 보조 정보는 오버레이/드로어로만 열고, 기본 편집 표면은 단일 패널을 유지한다.

### 5.2 Notebook (1024 ~ 1439)

- `Single Visible 모드`를 기본값으로 사용
- 현재 불편(속성 패널 아래 밀림) 발생 구간을 우선 개선
- Desktop과 동일한 상호작용 원칙을 사용한다.

### 5.3 Mobile/Tablet (< 1024)

- 고정 상단 세그먼트 대신 `좌상단 Items 버튼`으로 항목 선택 드로어/바텀시트를 연다.
- 본문은 단일 패널만 유지하고, 레일은 영구 고정하지 않는다.
- 기능 패리티 원칙: 선택/편집/저장/검증 오류 확인은 Desktop과 동일하게 제공한다.
- 버튼 사양: 리스트 아이콘 + `Items` 라벨(기본 노출).
- 표시 기준: 세로 방향(portrait)은 바텀시트, 가로 방향(landscape)은 좌측 드로어를 기본값으로 사용한다.
- 닫기 규칙: 모바일 뒤로가기/`Esc`/오버레이 클릭 시 Items 패널을 닫고, 본문 상태는 유지한다.

---

## 6) 접근성/사용성 기준

- 레일 항목은 버튼 역할과 명확한 라벨 제공
- 접힘 상태에서는 툴팁 + `aria-label` 보장
- 키보드 이동
  - `Up/Down`: 레일 항목 이동
  - `Enter/Space`: 항목 활성화
  - `Esc`: 열려 있는 오버레이/드로어 닫기
- 포커스 표시를 항상 명확히 유지

---

## 7) 성공 지표 (KPI)

### 7.1 정량 지표

1. 속성 패널 접근 시 평균 스크롤 횟수: `0회` 목표
2. 블록 선택 -> 첫 필드 입력까지 소요 시간: 기존 대비 `30% 단축`
3. 편집 세션 내 패널 전환 시간: 평균 `1초 이내`
4. 편집 중 불필요 스크롤 거리: 기존 대비 `20% 이상 감소`

### 7.2 정성 지표

- 운영자 인터뷰에서 "속성 패널 위치를 찾기 어렵다" 피드백 `0건`
- "편집 흐름이 끊긴다" 피드백 유의미 감소

---

## 8) 단계별 추진안 (기획 단계)

### Phase A: 상세 와이어 확정

- 레일 펼침/접힘 상태
- 단일 패널 전환 애니메이션
- 상태 유지 규칙(블록 선택/스크롤/미저장 값)

### Phase B: 인터랙션 명세 확정

- 자동 전환 정책(블록 클릭 시 속성 자동 이동 여부)
- 단축키/키보드 탐색 정책

### Phase C: 운영 검증

- 운영자 3~5명 태스크 기반 비교
- 현행 vs 개선안 수행시간/오류/피로도 측정

### Phase D: 구현 티켓 분해

- 레일 컴포넌트
- 단일 패널 상태 머신
- 반응형 정책
- QA 체크리스트

---

## 9) 비범위(Out of Scope)

이번 개선안에서 제외:

- 블록 타입 추가/삭제
- API 스키마 변경
- 권한 정책 변경
- 발행/버전 정책 변경

## 10) 한 줄 결론

"사이드 레일 + 단일 패널 노출" 구조로 바꾸면, 속성 패널 접근을 위해 스크롤하는 문제를 제거하고, 편집 집중도를 크게 올릴 수 있다.

## 11) 구현 티켓 분해 (Take2-U1~U4)

`docs/page_edit.md`의 U1~U4 구조와 동일하게, 본 문서의 개선안을 구현 착수 가능한 단위로 분해한다.

### 11.1 U1 레이아웃/상태 스펙 고정

- `Take2-U1-01` Side Rail 정보 구조 확정
  - 범위: 레일 항목(블록/캔버스/속성), 기본 활성 항목, 접기/펼치기 시각 규칙 확정
  - 산출물: 레일 상태 다이어그램(Expanded/Collapsed), 활성/비활성 스타일 표
  - DoD: 화면 크기별(Desktop/Notebook/Mobile) 레일 동작 모호성 0건

- `Take2-U1-02` Single Visible 패널 상태 전이 명세
  - 범위: 항목 전환, 자동 전환 옵션, 전환 중 상태 유지(선택 블록/입력값/스크롤)
  - 산출물: 상태 전이표 + 예외 시나리오(미저장 상태, 패널 전환, viewport 변경)
  - DoD: 전환 중 데이터 유실 시나리오 0건 + `PO 1명`과 `FE Lead 1명` 설계 검토 승인

### 11.2 U2 인터랙션 구현 티켓

- `Take2-U2-01` Side Rail 접기/펼치기 컨트롤
  - 범위: 상단 토글 버튼, 접힘 상태 아이콘 UI, 툴팁/aria-label
  - DoD: 마우스/키보드 모두에서 접기/펼치기 정상 동작

- `Take2-U2-02` 단일 패널 전환 컨테이너
  - 범위: 블록 목록/캔버스/속성 패널 중 1개만 렌더 또는 활성화
  - DoD: 전환 직후 첫 상호작용까지 1초 이내

- `Take2-U2-03` 컨텍스트 유지 로직
  - 범위: 패널 전환 전후 선택 블록, 입력 중 값, 캔버스 스크롤 복원

- `Take2-U2-04` 자동 전환 옵션

### 11.3 U3 QA/계측 티켓

- `Take2-U3-01` KPI 계측 이벤트 연결
  - 범위: 패널 접근 스크롤 횟수, 블록 선택->첫 필드 입력 시간, 패널 전환 시간, 편집 세션 스크롤 거리

  - DoD: KPI 4종(속성 패널 접근 스크롤 횟수, 블록 선택->첫 필드 입력 시간, 패널 전환 시간, 편집 세션 스크롤 거리)이 로그/대시보드에서 조회 가능

- `Take2-U3-02` 태스크 기반 비교 테스트
  - 범위: 현행 레이아웃 vs Take2 레이아웃 A/B 비교

- `Take2-U3-03` 접근성 회귀 QA
  - 범위: 키보드 탐색, 포커스 표시, aria-label/툴팁, Esc 닫기
  - DoD: 접근성 차단 이슈 0건

### 11.4 U4 롤아웃/운영 티켓

- `Take2-U4-01` 파일럿 롤아웃 정책
  - 범위: 내부 운영자 파일럿 그룹 선적용, Go/Hold/No-Go 기준

## 12) 고정 QA 체크리스트 (Release Gate)

체크 방식: 각 항목 `Pass/Fail/N.A.` 기록. `Fail` 1건 이상이면 릴리즈 보류.

`A. 레이아웃`

- [ ] Notebook(1024~1439)에서 속성 패널 접근을 위해 세로 스크롤이 필요하지 않다.
- [ ] Side Rail 접힘/펼침 전환 시 본문 패널 레이아웃이 깨지지 않는다.
- [ ] 단일 패널 전환 시 이전 패널 잔상/중첩 렌더링이 없다.

`B. 상태 유지`

- [ ] 블록 목록 -> 속성 패널 전환 시 선택 블록이 유지된다.
- [ ] 패널 전환 후에도 미저장 입력값이 유지된다.
- [ ] 캔버스 복귀 시 스크롤 위치가 복원된다.

`C. 접근성`

- [ ] 접힘 상태에서도 모든 레일 항목이 `aria-label`로 식별 가능하다.
- [ ] `Up/Down`으로 레일 항목 이동, `Enter/Space`로 활성화 가능하다.
- [ ] 오버레이/드로어 열린 상태에서 `Esc` 닫기가 동작한다.

`D. 성능/사용성`

- [ ] 패널 전환 직후 인터랙션 지연(체감 1초 초과)이 없다.
